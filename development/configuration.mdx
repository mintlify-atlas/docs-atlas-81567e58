|---|---|---|

## Expo plugins

### `expo-secure-store`

Built-in Expo plugin. Enables native keychain/keystore integration via `expo-secure-store` for storing secrets securely on device.

### `expo-video`

Built-in Expo plugin. Required for `expo-video` to configure platform-specific video playback entitlements.

### `withRemovePermissions` (custom)

Defined in `plugins/withRemovePermissions.js`. Uses `@expo/config-plugins` to add `tools:node="remove"` to specified Android permissions in the final `AndroidManifest.xml`. This prevents permissions from being merged in from transitive dependencies.

```javascript
// plugins/withRemovePermissions.js
const { withAndroidManifest } = require('@expo/config-plugins');

const withRemovePermissions = (config, permissions) => {
  return withAndroidManifest(config, async (config) => {
    // Adds xmlns:tools namespace if not present
    // Marks each permission with tools:node="remove"
    return config;
  });
};

module.exports = withRemovePermissions;
```

Currently used to remove `android.permission.RECORD_AUDIO`, which is pulled in by a transitive dependency but not needed by the app.

### `expo-build-properties`

Sets Android SDK version overrides in `app.json` without requiring a custom native project.

### Build profiles

<CardGroup cols={3}>
  <Card title="development" icon="code">
    Uses `expo-dev-client`. iOS builds target the simulator. Android builds include `EXPO_DEBUG=true`. Distribution is internal.
  </Card>
  <Card title="preview" icon="eye">
    Internal distribution. Android produces an APK (not AAB) for direct device testing. No iOS build is defined.
  </Card>
  <Card title="production" icon="rocket">
    `autoIncrement: true` — EAS increments the version code automatically. iOS targets a physical device. Android produces the default AAB.

`appVersionSource: "remote"` means EAS controls the authoritative version number; the local `app.json` version is a fallback.

## `babel.config.js`

```javascript
module.exports = function (api) {
  api.cache(true);
  return {
    presets: ['babel-preset-expo'],
    plugins: ['react-native-reanimated/plugin'],
  };
};
```

- **`babel-preset-expo`** — the standard Expo Babel preset, which includes `@babel/preset-typescript`, JSX support, and module resolution for Expo packages.
- **`react-native-reanimated/plugin`** — required by Reanimated 3; must be listed last in the plugins array. It transforms worklet functions at build time.

<Note>
  The Reanimated plugin must always be the last plugin in the Babel config. Moving it or adding plugins after it can cause runtime worklet errors.
</Note>

## `tsconfig.json`

```json
{
  "compilerOptions": {},
  "extends": "expo/tsconfig.base"
}
```

The project extends `expo/tsconfig.base`, which provides:

- `"strict": true`
- `"jsx": "react-native"`
- `"moduleResolution": "bundler"`
- Path aliases for Expo packages
- ES2022 target

No custom `compilerOptions` are set — the Expo defaults are used as-is. TypeScript files (`.ts`, `.tsx`) are supported in the project; the main entry (`App.js`) and most screens remain in JavaScript.
