## Security

```json
"bcryptjs":           "^2.4.3",
"expo-secure-store":  "~14.0.1",
"expo-crypto":        "~14.0.2"
```

- **bcryptjs** — imported as a dependency but the app uses a custom `secureHash()` function for password hashing (see [Architecture](/development/architecture))
- **expo-crypto** — cryptographic utilities from Expo (hashing, UUID)
- **expo-secure-store** — listed under Storage above; relevant here as a security primitive

<Note>
  `bcryptjs` is present in `package.json` but the active password hashing implementation uses the custom `secureHash()` function defined in `Src/Api/lib/api.ts`, `altaUsuario.ts`, `login.ts`, and `register.ts`. The `secureHash` format (`$secure$...`) is not bcrypt-compatible.
</Note>

## AI and chat

```json
"@google/generative-ai":   "^0.21.0",
"react-native-gifted-chat": "^2.6.4"
```

- **@google/generative-ai** — Google Gemini SDK included as a dependency; the active chatbot in `Chatbot.js` calls Stack AI via a direct `fetch()` to a Stack AI endpoint rather than this SDK
- **react-native-gifted-chat** — chat UI library included as a dependency; `Chatbot.js` uses a custom `FlatList`-based message renderer rather than GiftedChat's `GiftedChat` component directly

## `overrides`

The `overrides` field in `package.json` is present but empty (`{}`). No dependency version overrides are currently active.

<Info>
  The `eas-cli` package is listed under `dependencies` (not `devDependencies`) at version `^16.20.1`. For most workflows it is more conventional to install the EAS CLI globally. Having it in `dependencies` ensures CI environments use a pinned version without a global install.
</Info>
