## Key files explained

### `App.js` — Entry point

The root component. Manages three pieces of state (`isFirstLaunch`, `isLoggedIn`, `userSession`) and renders the appropriate Stack screen group based on the result of `checkAppState()`. All navigation in the app starts here.

### `Src/Api/lib/api.ts` — Canonical API client

The single source of truth for all backend communication. Exports `http()`, all CRUD functions, both type definitions (`UserRow`, `DegreeRow`), and all validation helpers. Other files in `Src/Api/` import from this module.

### `Src/Api/api.ts` — Extended debug client

A parallel implementation of `lib/api.ts` with additional `console.log` statements for development debugging. Used directly in some feature screens. Contains the same exported interface.

### `Src/Screens/Home/Components/SearchBarsComponent/MyDrawer.js`

Defines the Drawer Navigator with its custom content component. The drawer items are organized into four sections and each item maps a screen name to an imported screen component.

### `Src/Screens/Home/Components/BottonSheetComponent/Modulos/`

Contains one directory per campus building or location (Modulo_A through Modulo_Z2, plus Biblioteca, Cafebreria, Comida, Entradas, Jobs, Matute, Titanic, globo). Each module directory holds the content shown in the bottom sheet when that location is selected on the map.

## Naming conventions

| Pattern | Usage |
|---|---|
| `PascalCase` | Screen components (`LoginScreen`, `ProfileScreen`, `MyDrawer`) |
| `camelCase` | Utility functions (`findUserByEmail`, `cleanString`, `secureHash`) |
| `snake_case` | Legacy API wrappers (`alta_usuario`, `get_degrees`, `validar_correo`) |
| `UPPER_SNAKE_CASE` | Constants (`BASE`, `USE_INDEX_PHP`) |
| `Modulo_X` | Campus building directories, where X is a letter or code |
